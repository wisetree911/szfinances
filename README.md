# SZ Finances

Backend pet project для учёта и аналитики инвестиционного портфеля на данных Московской биржи.

Репозиторий я делал как backend-first проект вокруг реалистичного домена: пользователи, портфели, сделки, текущие цены активов, производные метрики, аутентификация и отдельный фоновый процесс обновления цен. Основная цель здесь не в демонстрационном интерфейсе, а в аккуратной инженерной сборке сервиса с понятными границами и тестируемой бизнес-логикой.

## Что Реализовано

- REST API на FastAPI для пользователей, активов, портфелей и сделок
- JWT-аутентификация с `access` / `refresh` токенами
- ротация `refresh` токена и его инвалидирование при logout
- разделение маршрутов на пользовательские и административные
- аналитика портфеля по истории сделок
- FIFO-расчёт оставшихся лотов и unrealized PnL
- сводка по портфелю: market value, cost basis, доходность и топ позиций
- аналитика распределения по секторам
- Redis-кэш для аналитических эндпоинтов
- отдельный модуль `price_updater`, который забирает цены из MOEX и сохраняет снапшоты в PostgreSQL
- миграции Alembic
- unit- и API-тесты

## Архитектура

- `backend/app` — основное API-приложение
- `backend/price_updater` — фоновый сервис обновления цен
- `backend/tests` — unit- и integration-тесты

Приложение разложено на `api`, `services`, `repositories`, `models` и `schemas`. HTTP-слой, бизнес-логика и работа с хранением разделены так, чтобы ключевые сценарии можно было тестировать отдельно от транспорта.

## Что Показывает Проект

- умение проектировать асинхронный backend
- работу с аутентификацией, авторизацией, миграциями и постоянным хранилищем
- расчёт финансовых метрик по истории сделок с FIFO, а не через упрощённые агрегаты
- внимание к структуре кода, тестируемости и воспроизводимому локальному запуску

Если смотреть код выборочно, самые показательные части здесь:

- `backend/app/services/analytics.py`
- `backend/app/analytics/analytics_calc.py`
- `backend/app/services/auth.py`
- `backend/tests/api`
- `backend/tests/unit`

## Локальный Запуск

```bash
cp .env.example .env
docker compose up --build
```

После старта Swagger доступен по адресу `http://localhost:8000/docs`.

Контейнер `backend` автоматически применяет Alembic-миграции при запуске.

По умолчанию `docker compose` поднимает:

- API
- PostgreSQL
- Redis

Модуль `price_updater` уже есть в репозитории, но пока не подключён в дефолтный сценарий `docker compose` как отдельный сервис.

## Проверка

```bash
cd backend
make check
```

Для API-тестов:

```bash
cd backend
make test-api-up
make test-api
make test-api-down
```

## Текущее Состояние

Это backend-first pet project. Frontend-клиента в репозитории пока нет.

Realtime-часть частично подготовлена через Redis и WebSocket-инфраструктуру, но end-to-end сценарий ещё не доведён до конца.
